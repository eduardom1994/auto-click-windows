# Auto-Clicker Project Instructions

## Project Overview
- Python application for automated clicking with GUI.
- Uses Tkinter for interface, pyautogui for mouse control, pynput for hotkeys.
- Supports multiple locations with custom intervals.
- Compiled to standalone Windows exe using PyInstaller.

## Key Conventions
- **Thread Safety**: Always use `threading.Lock` in methods like `add_location` to prevent race conditions and duplicates.
- **Testing**: Run unit tests (`pytest test_main.py`) after code changes; interface tests (`test_interface.py`) for GUI validation.
- **Compilation**: After changes, compile with `pyinstaller --onefile --noconsole main.py` and test the exe.
- **Versioning**: Use Git for commits; push to main branch after validation.
- **Error Handling**: Use `messagebox` for user feedback; validate inputs (e.g., positive floats for intervals).

## Common Tasks
- **Adding Features**: Include UI buttons, validation, and tests. Update README.md.
- **Bug Fixes**: Focus on threading issues (duplicates, pauses); add cleanup buttons (e.g., remove duplicates).
- **UI Improvements**: Expand grid for new buttons; ensure labels span columns.
- **Dependencies**: Install via pip in venv; list in requirements.txt.

## Workflow
1. Edit code in `main.py`.
2. Run tests.
3. Compile exe.
4. Commit and push.

## Dependencies
- pyautogui
- pynput
- tkinter (built-in)

## Testing
- Unit: `test_main.py` (pytest)
- Interface: `test_interface.py` (simulates GUI)
